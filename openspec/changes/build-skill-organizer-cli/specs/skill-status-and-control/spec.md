## ADDED Requirements

### Requirement: The system SHALL allow source skills to be enabled and disabled
The system SHALL provide commands that update `metadata.skill-organizer.disabled` in the source `SKILL.md` file and sync behavior SHALL exclude disabled skills from the managed target.

#### Scenario: Disable marks the source skill as disabled
- **WHEN** the user disables a source skill by its source path
- **THEN** the system writes `metadata.skill-organizer.disabled: true` in that source skill's `SKILL.md`

#### Scenario: Disabled skills are absent from the target after sync
- **WHEN** sync runs for a source skill whose `metadata.skill-organizer.disabled` is `true`
- **THEN** the system does not keep a managed target symlink for that skill

### Requirement: Status SHALL report managed and unmanaged target state
The system SHALL provide a status command that lists discovered source skills, their disabled state, managed target sync state, and target entries that are not generated by the sync tool.

#### Scenario: Status shows disabled source skills distinctly
- **WHEN** a source skill is marked disabled in `metadata.skill-organizer`
- **THEN** the status output identifies that skill as disabled and shows that it is not expected in the target

#### Scenario: Status shows unmanaged target entries
- **WHEN** the target contains entries that are not managed by the sync tool
- **THEN** the status output lists those entries separately from managed sync results

### Requirement: The system SHALL move unmanaged target skills into the organized source tree on confirmation
The system SHALL provide an interactive command that previews moving unmanaged target skill folders into the organized source tree and SHALL require confirmation before applying the move unless a non-interactive override is provided.

#### Scenario: Move unmanaged previews before execution
- **WHEN** the user runs the move-unmanaged command interactively and unmanaged target entries exist
- **THEN** the system presents the proposed moves and asks for confirmation before changing files
